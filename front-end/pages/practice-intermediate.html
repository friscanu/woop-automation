Docker Container Workshop

Intermediate Hands-On
Estimated Time: 90 minutes
Tasks: 6

Tasks

  • 1 Docker Installation Check

    Verify that Docker is installed and running correctly:

    Try the following:

    1. Use docker --version to check the Docker version
    2. Use docker info to view system-wide information
    3. Use docker run hello-world to verify the installation
  • 2 Working with Docker Images

    Learn how to pull, list, and inspect Docker images:

    Try the following:

    1. Use docker pull nginx:latest to download the Nginx web server image
    2. Use docker images to list all downloaded images
    3. Use docker image inspect nginx:latest to view detailed image information
    4. Use docker search ubuntu to search for Ubuntu images on Docker Hub
  • 3 Running Docker Containers

    Learn how to run, manage, and interact with containers:

    Try the following:

    1. Use docker run -d -p 8080:80 --name my-nginx nginx:latest to start an Nginx container
    2. Use docker ps to list running containers
    3. Use curl http://localhost:8080 to verify the web server is working
    4. Use docker logs my-nginx to view container logs
    5. Use docker exec -it my-nginx bash to access a shell in the container
  • 4 Container Lifecycle Management

    Learn how to stop, start, and remove containers:

    Try the following:

    1. Use docker stop my-nginx to stop the running container
    2. Use docker ps -a to list all containers (including stopped ones)
    3. Use docker start my-nginx to restart the container
    4. Use docker restart my-nginx to stop and start in one command
    5. Use docker rm -f my-nginx to remove the container
  • 5 Creating a Dockerfile

    Learn how to create a custom Docker image using a Dockerfile:

    Try the following:

    1. Create a directory: mkdir mydockerapp && cd mydockerapp
    2. Create a file named index.html with the content: echo '

      My Custom Docker Image

      ' > index.html
    3. Create a Dockerfile: echo -e 'FROM nginx:latest\nCOPY index.html /usr/share/nginx/html/index.html' > Dockerfile
    4. Build the image: docker build -t my-custom-nginx .
    5. Run the custom image: docker run -d -p 8081:80 my-custom-nginx
    6. Verify with: curl http://localhost:8081
  • 6 Docker Networking

    Learn how to work with Docker networks:

    Try the following:

    1. List networks: docker network ls
    2. Create a new network: docker network create my-network
    3. Run a container on the network: docker run -d --name web1 --network my-network nginx:latest
    4. Run another container: docker run -d --name web2 --network my-network nginx:latest
    5. Connect to a container and ping the other: docker exec -it web1 bash -c "apt-get update && apt-get install -y iputils-ping && ping web2"
    6. Clean up: docker rm -f web1 web2 && docker network rm my-network

Lab Instructions

This workshop will teach you how to work with Docker containers, a popular technology for application deployment and management. You'll learn to create, manage, and network containers using Docker commands.

Getting Started

  1. Work through the tasks in order, using the Docker terminal to the right.
  2. Each task builds on the previous one, demonstrating different aspects of Docker.
  3. The terminal is pre-configured with Docker and all necessary tools installed.
  4. Use the tabs below for additional information and resources.

Completion Requirements

To successfully complete this lab, you must finish all 6 tasks. Each task will be marked as complete when you've successfully executed the required Docker commands.

Important Note

Remember to clean up containers and networks after you're done with them to avoid resource conflicts. The commands for cleanup are provided in the task instructions.

Helpful Hints

Container Basics

  • A container is a lightweight, standalone, executable package that includes everything needed to run an application
  • Unlike VMs, containers share the host OS kernel but have isolated file systems
  • The -d flag runs containers in detached mode (background)
  • The -p flag maps host ports to container ports (e.g., -p 8080:80)

Dockerfile Tips

  • Each instruction in a Dockerfile creates a new layer in the image
  • Use FROM to specify the base image
  • Use COPY to add files from your host to the image
  • Use RUN to execute commands during the build
  • Use CMD to specify the command to run when the container starts

Common Commands

  • Use docker ps -a to show all containers (including stopped ones)
  • Use docker rm $(docker ps -aq) to remove all containers
  • Use docker rmi $(docker images -q) to remove all images
  • Use docker exec -it [container] [command] to run commands in a running container
